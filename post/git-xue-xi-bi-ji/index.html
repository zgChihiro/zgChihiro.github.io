git学习笔记

Posted on 2017-03-23，3 min read

工作区 >> 暂存区 >> 分支
leangit >> stage(index) >>master

配置git信息
**git config --global user.name
git config --global user.email **

ssh-keygen -t rsa -b 4096 -C "your_email@example.com"电脑生成钥匙
cat ~/.ssh/id_rsa.pub 查看本地钥匙

ssh复制到粘贴版
↓↓↓↓↓
clip < ~/.ssh/id_rsa.pub

文件添加仓库的两步

mkdir 创建文件夹

git init 将目录编程git可以管理的仓库

git add readme.txt
git commit -m "提示 或者时间"
git remote add origin git@github.com:q3689852/仓库名字.git本地库上传github

pwd 显示当前目录

git status 查看仓库状态（文件是否被修改）
git diff 查看文件修改的不同（别修改内容）工作区和暂存区的区别

git log 查看之前的版本
git log --pretty=oneline 上者的简约版

cat readme.txt 可以展示出此文件的内容

git reset --hard HEAD^ 版本回滚^表示一个版本多个用~加数字
git reset --hard 3628164 加版本号的前几位可任意滚

git checkout -- readme.txt 撤销修改工作区的文件
git reset HEAD -- readme.txt 暂存区的修改撤销掉，重新放回工作区

git reflog 列出版本滚动记录

rm readme.txt 删除文件

git clong 链接 下载项目到本地

git branch 列出本地分支
git branch VR_17_dgz 新建分支
git checkout -b VR_17_dgz 新建分支屏切换到当前分支
git checkout VR_17_dgz 跳转到此分支
git checkout - 切换到上一个分支

将新分支发布在github上： git push origin Branch1
在本地删除一个分支： git branch -d Branch1
在github远程端删除一个分支： git push origin :Branch1   (分支名前的冒号代表删除)

git merge name 将name合并到当前分支(自身被覆盖)
git merge --no-ff -m "TiShi" name 和上着的区别是会多产生一个提交
git log --graph 查看分支合并图

git stash 隐藏工作现场/暂停
git stash list 查看被隐藏的列表
git stash pop 恢复隐藏

git remote add origin git@github.com:q3689852/库名.git
git push -u origin master

git checkout --文件夹或文件 恢复工作区

大文件的解决方法
下载安装LFS
https://git-lfs.github.com/
git lfs install 初始化
git lfs track“* .psd” 大文件

ssh-keygen -t rsa -b 4096 -C "your_email@example.com"电脑生成钥匙
cat ~/.ssh/id_rsa.pub 查看本地钥匙
